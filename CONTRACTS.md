### SSE protocol for `POST /api/agent/chat`
Request `{"message": str, "chat_id": str|null}`. Events:
- `event: token`            data: `{"text": str}`
- `event: tool`             data: `{"name": str, "args": obj}`
- `event: table_created`    data: `{"table": {id, slug, name, columns, dedupe_keys}}`  **(new in v2)**
- `event: workflow_created` data: `{"workflow": {id, name, dsl}}`
- `event: run_started`      data: `{"run_id": int, "workflow_id": int}`
- `event: done`             data: `{"chat_id": str}`

## Workflow DSL v2 (`pipelines/dsl.py::validate_dsl(dsl, valid_sources=None) -> list[str]`, empty = valid)
```json
{ "name": str, "trigger": "manual", "table": "<table slug — MUST exist>",
  "steps": [
    {"type": "fetch",   "source": "whatsapp"|"gmail", "since_days": int?, "from_date": iso?, "to_date": iso?, "chat_jids": [str]?},
    {"type": "filter",  "instruction": str},
    {"type": "extract"},
    {"type": "upsert",  "dedupe_on": [str]?}
  ] }
```
Rules: `table` required and must resolve to an existing DataTable; ≥1 fetch (source must be
in the connector registry); filter optional; extract + upsert required; **`extract` takes NO
fields — the target table's `columns` ARE the field list**; `upsert.dedupe_on` defaults to
the table's `dedupe_keys` (empty ⇒ always insert); unknown step types are errors.

Engine (`pipelines/engine.py::start_run`): thread; per-message extract typed against the
table schema; upsert matches an existing Record where ALL dedupe key values are equal
(compare as str); merge fills null/missing fields; provenance appended to `sources`.
`run.stats = {"fetched": n, "kept": n, "rows_created": n, "rows_updated": n, "table": slug}`.

## Extractor (`pipelines/extractor.py`)

  connectors/*.yaml   declarative connector registry (read-only)
