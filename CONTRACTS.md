## REST API (all under `/api/`, DRF, no auth, JSON)
- `GET /api/contacts/` — list; supports `?search=` (name/email/phone/company icontains); each item includes `interaction_count` and `last_ts` annotations
- `GET /api/contacts/{id}/` — detail, includes nested `interactions` (ordered by ts desc)
- `GET /api/interactions/?contact={id}`
- `GET /api/workflows/`, `POST /api/workflows/` (`{name, dsl}`)
- `POST /api/workflows/{id}/run/` → `202 {"run_id": int}`
- `GET /api/runs/`, `GET /api/runs/{id}/` → `{id, workflow, status, stats, log, started_at, finished_at}`
- `GET /api/connections/`
- `POST /api/connections/whatsapp/pair/` → proxies sidecar `GET /qr` → `{qr, connected}`
- `POST /api/connections/gmail/connect/` → `{status, redirect_url?}` (mock: sets status=connected)
- `POST /api/agent/chat` — **SSE stream** (see below)
- `POST /api/ingest/whatsapp` — header `X-Ingest-Secret: <INGEST_SECRET>`; body `{"messages": [IngestMessage]}` → `{"created": int, "skipped": int}`; dedupe on (source, external_id)

### IngestMessage shape (sidecar → Django, and fixture file shape)
```json
{"external_id": "wa-...", "chat_jid": "9198...@s.whatsapp.net", "sender_name": "Asha Rao",
 "phone": "+919812345678", "body": "Hi, what's the pricing for the pro plan?",
 "ts": "2026-07-10T14:22:00Z", "direction": "in"}
```
Gmail fixture shape: `{"external_id": "gm-...", "sender_name": str, "email": str, "subject": str, "body": str, "ts": iso, "direction": "in"|"out"}`

### SSE protocol for `POST /api/agent/chat`
Request JSON: `{"message": str, "chat_id": str|null}`. Response `text/event-stream`, events:
- `event: token`            data: `{"text": str}` — assistant text chunk
- `event: tool`             data: `{"name": str, "args": obj}` — tool call started
- `event: workflow_created` data: `{"workflow": {id, name, dsl}}`
- `event: run_started`      data: `{"run_id": int, "workflow_id": int}`
- `event: done`             data: `{"chat_id": str}`

## Agent tools (agentcore/tools.py — names and signatures fixed)
- `list_sources()` → `{"sources": [{"source", "status", "mode": "live"|"mock"}]}`
- `create_workflow(name: str, dsl: dict)` → validates via `pipelines.dsl.validate_dsl`, saves Workflow → `{"workflow_id", "name", "dsl"}`
- `run_workflow(workflow_id: int)` → starts engine thread → `{"run_id"}`
- `query_crm(question: str)` → simple stats `{"contacts": int, "interactions": int, "by_source": {...}, "recent_tags": [...]}`

## Workflow DSL (validated by `pipelines/dsl.py::validate_dsl(dsl) -> list[str]` returning error list, empty = valid)
```json
{ "name": str, "trigger": "manual",
  "steps": [
    {"type": "fetch",   "source": "whatsapp"|"gmail", "since_days": int},
    {"type": "filter",  "instruction": str},
    {"type": "extract", "fields": [str, ...]},
    {"type": "upsert",  "dedupe_on": ["phone","email"], "tag": str|null}
  ] }
```
Rules: ≥1 fetch required; filter optional; extract+upsert required; unknown step types are errors.
Engine: `pipelines/engine.py::start_run(workflow) -> WorkflowRun` — runs interpreter in `threading.Thread`, appends lines to `run.log`, sets `run.stats` = `{"fetched": n, "kept": n, "contacts_created": n, "contacts_updated": n, "interactions_created": n}`.
Upsert dedupe: match existing Contact by phone OR email (first hit wins); merge fills null fields, appends tag if not present.

## Hermes client (agentcore/hermes_client.py)
- `get_client()` → returns `HermesClient` if `HERMES_API_KEY` set, else `MockHermesClient`. Both expose:
  - `chat(messages: list, tools: list) -> iterator of {"type": "text", "text"} | {"type": "tool_call", "name", "args"}`
  - `extract(body: str, fields: list[str]) -> dict` (structured output)
  - `filter_relevant(bodies: list[str], instruction: str) -> list[bool]`
- MockHermesClient behavior: `chat` recognizes intent keywords (whatsapp/gmail/import/pricing) and emits a plausible plan narration + `create_workflow` tool call with a valid DSL + `run_workflow` after creation confirmation; `extract` uses regex/heuristics (phone/email patterns, name from sender, naive intent keyword); `filter_relevant` keyword-matches the instruction. Mock must produce a convincing demo transcript.
